# Contributing to Enhancer for Gemini

First off, thank you for considering contributing to **Enhancer for Gemini**! It's people like you that make open-source software such a great community.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/berkeguvenc/gemini-booster/issues) to see if it has already been reported. If not, feel free to open a new issue.

## Development Setup

This project uses the [Plasmo](https://docs.plasmo.com/) framework, React, and TypeScript. We use `pnpm` as our package manager.

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/gemini-booster.git
   ```
3. **Install dependencies**:
   ```bash
   cd gemini-booster
   pnpm install
   ```
4. **Run the development server**:
   ```bash
   pnpm run dev
   ```
   This will start the Plasmo dev server and create a local development build that you can load into your browser.

## Making Changes

- Create a new branch for your feature or bugfix (`git checkout -b feature/my-awesome-feature`).
- Ensure your code follows the existing style and conventions.
- Make your changes and test them locally to ensure everything works smoothly in the Gemini interface.

## Submitting a Pull Request

1. **Commit your changes** with a clear and descriptive commit message.
2. **Push to your branch** (`git push origin feature/my-awesome-feature`).
3. **Open a Pull Request** against the `main` branch of the original repository.
4. Please provide a clear description of the changes you made and any relevant issue numbers (e.g., "Fixes #12").
